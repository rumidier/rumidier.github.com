Less Tutorial

Nov 21st, 2012 | Comments

Tutorial

Variables(변수)

변수 선언은 ‘@’ 연산자를 사용 합니다. 변수에 값을 정의 함으로써 광범위 하게 사용 할수 있습니다.

example.less

@color: #4D926F

#header {
  color: @color;
}
h2 {
  color: @color;
}

compile.css

.css로 변환 하면 @color의 값으로 치환됩니다.

 #header {
   color: #4D926F;
 }
 h2 {
   color: #4D926F;
 }

Mixins(믹스 인)

믹스인은 클래스 처럼 사용 가능 하며 클래스에서 설정한 모든 속성을 포함시킬수 있습니다. 또한, 함수와 같이 인수를 설정 할수도 있습니다.

example.less

.rounded-corners (@radius: 5px) {
  -webkit-border-radius: @radius;
  -moz-border-radius: @radius;
  -ms-border-radius: @radius;
  -o-border-radius: @radius;
  border-radius: @radius;
}

#header {
  .rounded-corners;
}
#footer {
  .rounded-corners(10px);
}

compile.css

.rounded-coners를 선언 하였고 기본 인수는 @radius 5px입니다. #header는 기본 인수를 사용 하였고 #footer는 사용자 인수를 사용 하여서 10px이 되었습니다.

.rounded-corners (@radius: 5px) {
  -webkit-border-radius: @radius;
  -moz-border-radius: @radius;
  -ms-border-radius: @radius;
  -o-border-radius: @radius;
  border-radius: @radius;
}

#header {
  -webkt-boder-radius: 5px;
  -moz-border-radius: 5px;
  -ms-border-radius: 5px;
  -o-border-radius: 5px;
  border-radius: 5px;
}
#footer {
  -webkit-border-radius: 10px;
  -moz-border-radius: 10px;
  -ms-border-radius: 10px;
  -o-border-radius: 10px;
  border-radius: 10px;
}

Nested Rules(중첩 규칙)

선택기를 중첩하여 상속 할수 있으므로 긴 이름의 선택기를 중복으로 사용하지 않고도 명확하고 짧게 사용할수 있습니다.

example.less

#header {
  h1{
    font-size: 26px;
    font-weight: bold;
  }
  p {
    font-size: 12px;
    a {
      text-decoration: none;
      &:hover { border-width: 1px }
    }
  }
}

compile.css

.less 에서 쓰인 한번 쓰인 #heder 선택기가 중복으로 적용됩니다.

#header h1 {
  font-size: 26px;
  font-weight: bold;
}    
#header p {
  font-size: 12px;
}
#header p a {
  text-decoration: none;
}
#header p a:hover { border-width: 1px }

Functions & Operations(함수와 연산)

연산은 색이나 숫자에 대해 덧셈, 뺄셈, 나눗셈, 곱셈을 할수 있습니다. 속성 사이에 복잡한 연산은 CSS의 호환성을 위해 괄호 내에서만 실행 됩니다.

example.less

@the-border: 1px;
@base-color: #111;
@red: #842210;

#header {
  color: @base-color * 3;
  border-left: @the-border;
  border-right: @the-border * 2;
}
#footer {
  color: @base-color + #003300;
  border-color: desaturate( @red, 10%);
}

compile.css

#header {
  color: #333;
  border-left: 1px;
  border-right: 2px;
}    
#footer {
  color: #114411;
  border-color: #7d2717
}

The Language

Variables(변수)

LESS에서 변수는 실제로 상수로 기억되며 변수는 오직 한번만 정의 됩니다.

example-01.less

@nice-blue 변수를 선언하고 + 연산을 합니다.

@nice-blue: #5B83AD;
@light-blue: (@nice-blue + #111);

#header { color: @light-blue; }

example-01.css

#header { color: #6c94be; }

example-02.less - 변수에 다른 변수가 정의 가능합니다.

@var에 문자열 ‘fnord’는 @fnord로 치환 됩니다.

@fnord: "I am fnord";
@var: 'fnord';
content: @@var;

example-02.css

content: "I am fnord"

Mixins

Mixin은 이름을 사용해 선언 할수 있고 클래스 처럼 동작 합니다.

example-03.less

.bordered {
  border-top: dotted 1px black;
  border-bottom: solid 2px black;
}

example-03.less

클래스 이름을 사용해 속성을 포함 시킬수있습니다.

.bordered {
  border-top: dotted 1px black;
  border-bottom: solid 2px black;
}

#menu a {
  color: #111;
  .bordered;
}
.post a {
  color: red;
  .bordered;
}

example-03.css

선언된 .bordered는 .css에서 보이지 않으며 치환 된채로 컴파일 됩니다.

#menu a {
  color: #111;
  border-top: dotted 1px black;
  border-bottom: solid 2px black;
}
.post a {
  color: red;
  border-top: dotted 1px black;
  border-bottom: solid 2px black;
}

Parametric Mixins

example-04.less

클래스 처럼 믹스인 할수 있으며, 매개변수를 전달할수 있습니다.

.border-radius (@radius) {
  border-radius: @radius;
  -moz-border-radius: @radius;
  -webkit-border-radius: @radius;
}

#header {
  .border-radius(4px);
}
.button {
  .border-radius(6px);
}

example-04.css

.button의 6px, #header에 4px 값을 전달 받아 선언합니다.

#header {
  border-radius: 4px;
  -moz-border-radius: 4px;
  -webkit-border-radius: 4px;
}
.button {
  border-radius: 4px;
  -moz-border-radius: 4px;
  -webkit-border-radius: 4px;
}

example-05.less

매개 변수가 유효한 mixin은 기본값을 가질수 있습니다

.border-radius (@radius: 5px) {
  border-radius: @radius;
  -moz-border-radius: @radius;
  -webkit-border-radius: @radius;
}
#header {
  .border-radius;
}

example-05.css

매개 변수 전달 없이 기본값을 사용 합니다.

#header {
  border-radius: 5px;
  -moz-border-radius: 5px;
  -webkit-border-radius: 5px;
}

example-06.less

.wrap () {
    text-wrap: wrap;
    white-space: pre-wrap;
    white-space: -moz-pre-wrap;
    word-wrap: break-word;
}
pre { .wrap }

example-06.css

pre {
  text-wrap: wrap;
  white-space: pre-wrap;
  white-space: -moz-pre-wrap;
  word-wrap: break-word;
}

The @arguments varialbe

@arguments를 사용해서 개별적으로 파라미터를 사용 하지 않아도 됩니다.

example-07.less

2px, 5px 두개의 인수를 전달 하면 순서대로 @x, @y에 적용 됩니다.

.box-shadow (@x: 0, @y: 0, @blur: 1px, @color: #000) {
  box-shadow: @arguments;
  -moz-box-shadow: @arguments;
  -webkit-box-shadow: @arguments;
}

.box-shadow(2px, 5px);

example-07.css

0, 0값에서 전달된 2px, 5px값으로 치환 되었습니다.

.box-shadow: 2px 5px 1px #000;
-moz-box-shadow: 2px 5px 1px #000;
-webkit-box-shadow: 2px 5px 1px #000;

Pattern-matching and Guard expressions

mixin을 선언하고 조건문으로 인자를 비교해 같은 이름의 .mixin중에 참인 .mixin을 적용 합니다.

변경 기본 문법

basic-01.less

.mixin (@s, @color) {
    ….
}
.class {
  .mixin(@switch, #888);
}

basic-02.less

.mixin (dark, @color) {
  color: darken(@color, 10%);
}
.mixin (light, @color) {
  color: lighten(@color, 10%);
}
.mixin (@), @color) {
  display: block;
}

example-08.less

.mixin이 3개가 선언 되었고 각 다른 조건문을 가지고 있습니다.

.mixin (dark, @color) {
  color: darken(@color, 10%);
}
.mixin (light, @color) {
  color: lighten(@color, 10%);
}
.mixin (@_, @color) {
  display: block;
}

@switch: light;
.class {
  .mixin(@switch, #888);
}

example-08.css

호출시 조건문 light값을 충족하며 모든 값을 의미 하는 ‘@_’포함 하여 적용 됩니다.

.class {
  color: #a2a2a2;
  display: black;
}

조건문으로 인수의 개수에 따라서 선택 할수도 있습니다.

.mixin (@a) {
  color: @a;
}
.mixin (@a, @b) {
  color: fade(@a, @b);
}

Guards

가드는 값이나 변수의 개수가 아니라 식에 일치 시킵니다.

example-09.less

if/else 대신 when을 사용 합니다. @a값을 비교하여 적용 하며 인자가 하나이므로 color는 전체에 적용 됩니다.

.mixin (@a) when (lightness(@a) >= 50%) {
  background-color: black;
}
.mixin (@a) when (lightness(@a) < 50%) {
  background-color: white;
}
.mixin (@a) {
  color: @a;
}
.class1 { .mixin(#ddd) }
.class2 { .mixin(#555) }

example-09.css

값에 비교에 따른 적용과 전체 적용된 color 입니다.

.class1 {
  color: #ddd;
  background-color: black;
}
.class2 {
  color: #555;
  background-color: white;            
}

가드 안에서는 비교식(>, >=, =, =<, <)을 사용 할수있습니다.

basic.less

두개의 식은 같은 표현 입니다.

.truth (@a) when (@a) { … }
.truth (@a) when (@a = true) { … }

example-10.less

아래의 식은 위에 basic.less값에 매치 되지 않습니다. 값이 ‘true’만 매치 됩니다.

.class {
  .truth(40);
}

example-11.less

‘or’ 이중 조건은 ‘,’로 분류 됩니다. 작성된 조건의 범위를 벗어난 값을 호출시 컴파일 자체가 되지 않습니다.

.mixin (@a) when (@a > 10), (@a < -10) { … }

example-12.less

인자의 값이 동등 한지 비교 가능 하며 대,소 또한 가능 합니다.

@media: mobile;

.mixin (@a) when (@media = mobile) { … }
.mixin (@a) when (@media = desktop) { … }

.max (@a, @b) when (@a > @b) { width: @a }
.max (@a, @b) when (@a < @b) { width: @b }

매개변수가 number 인지 color인지 비교 할수 있습니다.

.mixin (@a, @b: 0) when (isnumber(@b)) { … }
.mixin (@a, @b: black) when (iscolor(@b)) { … }

확인 가능한 is*형

  • iscolor
  • isnumber
  • isstring
  • iskeyword
  • isurl
  • ispixel
  • ispercentage
  • isem

추가 조건문 and는 숫자인지 비교하고 0보다 높은 값이어야 합니다.

.mixin (@a) when (isnumber(@a)) and (@a > 0) { … }

부정 조건문 not 조건이 0보다 크지 않을때 동작 합니다.

.mixin (@b) when not (@b > 0) { … }

조건문

Less

  • = : 두 값이 같으면 참입니다.
  • not : 두 값이 같지 않으면 참입니다.
  • and : 모든 조건이 참이어야 합니다.
  • , : 모든 조건 값중에 하나만 참이면 참입니다.

Nested rules

example-13.css

& 콤비네이터를 사용한 상속을 통해서 간결하게 만들수 있습니다. 아래는 .css파일 입니다. 이를 .less로 바꾸어 보겠습니다.

#header { color: black; }
#header .navigation {
  font-size: 12px;
}
#header .logo {
  width: 300px;
}
#header .logo:hover {
  text-decoration: none;
}

example-13-01.less

.css에서 반복 되던 #header를 한번만 선언 할수 있습니다.

#header {
  color: black;

  .navigation {
    font-size: 12px;
  }
  .logo {
    width: 300px;
    &:hover { text-decoration: none; }
  }
}

example-13-02.less

위와 결과는 같지만 더 간결한 문법을 보입니다.

#header { color: black;
  .navigation { font-size: 12px }
  .logo       { width: 300px;
    &:hover   { text-decoration: none }
  }  
}

example-14.less

& 콤비네이터를 사용한 또다른 형태 입니다.

.bordered {
  &.float { flat: left; }
  .top {
    margin: 5px;
  }
}

example-14.css

.bordered.float {
  flat: left;
}
.bordered.top {
  margin: 5px;
}

Advanced Usage of &

example-15.less

& 또다른 사용 법입니다. 모든 셀렉터의 조합을 만들어 줄수 있습니다.

.child, .sibling {
  .parent & {
    color: black;
  }
  & + & {
    color: red;
  }
}

example-15.css

.parent.child, .parent.sibling {
  color: black;
}

.child + .child,
.child + .sibling,
.sibling + .child,
.sibling + .sibling {
  color: red;
}

Operations

example-16.less

숫자, 색, 변수 또한 연산을 할수 있습니다.

@base: 5%;
@filler: (@base * 2);
@other: (@bae + @filler);

color: (#888 / 4);
background-color: (@base-color + #111);
height: (100% / 2 + @filler);

example-16-01.less

@var: (1px + 5); // @var: 6px

example-16-02.less

width: ((@var + 5) * 2);

Color functions

  • 밝기 조절

    lighten(@color, 10%); darken(@color, 10%);

  • 채도 조절

    saturate(@color, 10%); desaturate(@color, 10%);

  • 투명도

    fadein(@color, 10%); fadeout(@color, 10%); fade(@color, 50%);

  • 색장 조절

    spin(@color, 10); spin(@color, -10);

  • @color1, 과 @color2 혼합된 값을 반환 합니다. @weight는 기본값을 의미 합니다.

    mix(@color1, @color2, @weight);

  • @color1 > 50% luma 일경우 @darkcolor 를 아니면 @lightcolor를 반환 합니다. contrast(@color1, @darkcolor, @lightcolor;);

exampl1-17.less

@base: #f04615;

.class {
  color: saturate(@base, 5%);
  background-color: spin(lighten(@base, 25%), 8);
}

색정보 알아내기

hue(@color);
saturation(@color);
lightness(@color);
red(@color);
green(@color);
blue(@color);
alpha(@color);
luma(@color);

새로운 색 구하기

@new는 @old의 색을 유지하면서 자신의 채도 및 밝기를 정의

@new: hsl(hue(@old), 45%, 90%);

Namespaces

example-18.less

Namespace를 가지고 Mixin 에서 선택 할수 있습니다. 다음은 #bundle 에서 .button을 Mixin 하는 방법 입니다.

#bundle {
  .button () {
    display: block;
    border: 1px solid black;
    background-color: grey;
    &:hover { background-color: white }
  }
  .tab { … }
  .citaion { … }
}

#header a {
  color: orange;
  #bundle > .button;
}

compile.css

.tab, .citaion은 적용 되지 않습니다.

#header a {
  color: orange;
  display: black;
  border: 1px solid black;
  background-color: grey;
}
#header a:hover {
  background-color: #ffffff;
}
